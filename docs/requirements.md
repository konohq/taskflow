# TaskFlow AI 要件定義

## 目的

TaskFlow AI は、チームがプロジェクトとタスクを管理するための Web アプリです。

MVP では、認証、認可、チーム単位のデータ分離、Project / Task / Comment 管理、カンバン表示、自分の担当タスク一覧、自分が作成したタスク一覧を扱います。バックエンドは Rails API、フロントエンドは React SPA として実装し、Rails API から取得した実データを画面に表示します。

将来的には AI によるタスク分解、進捗要約、今日やるべきタスクの提案などを追加しますが、MVP ではチームタスク管理の基本機能を優先します。

## MVP の対象ユーザー

- チームで作業する一般ユーザー
- チームを作成してメンバーを管理するユーザー
- プロジェクトとタスクを管理するユーザー
- 自分に割り当てられたタスクを確認したいユーザー

## MVP の対象範囲

### 認証

- ユーザーは名前、メールアドレス、パスワードで登録できる
- ユーザーはメールアドレスとパスワードでログインできる
- ログイン中のユーザーはログアウトできる
- ログイン中のユーザー情報を取得できる
- 未ログインユーザーは認証が必要な API と画面にアクセスできない
- 認証には devise-jwt を使用する
- 認証が必要な API は Authorization ヘッダーの Bearer Token を検証する
- フロントエンドは取得した token を API 通信時に Authorization ヘッダーへ付与する
- セッション Cookie 認証は使用しない

### チーム

- ログインユーザーはチームを作成できる
- owner はチームを更新・削除できる
- チーム作成者はチームの owner になる
- ログインユーザーは自分が所属するチーム一覧を取得・表示できる
- ログインユーザーは自分が所属するチームの詳細を取得・表示できる
- チームに所属していないユーザーは、そのチームの情報を取得できない

### チームメンバー

- owner または admin はチームメンバーを追加できる
- owner のみチームメンバーの権限を変更できる
- admin はチームメンバーの権限を変更できない
- 権限変更は admin / member 間のみ許可し、owner 自身の権限変更と owner への権限変更はできない
- owner または admin はチームメンバーを削除できる
- member はチームメンバー一覧を閲覧できる
- 他チームのメンバー情報にはアクセスできない
- MVP では既存ユーザーをメールアドレスでチームに追加する
- 招待メール送信、招待リンク、承認フローは MVP 対象外とする
- チームメンバー削除時、そのユーザーが担当していた対象チーム内のタスクは削除せず、assignee_id を null にする
- フロントエンドではメンバー一覧表示とメンバー追加 UI を実装する
- TeamMember の role 変更・削除 UI は後回しにする

### プロジェクト

- チームメンバーは所属チーム内にプロジェクトを作成できる
- チームメンバーは所属チームのプロジェクト一覧を取得・表示できる
- チームメンバーは所属チームのプロジェクト詳細を取得・表示できる
- チームメンバーは所属チームのプロジェクトを更新・削除できる
- 他チームのプロジェクトにはアクセスできない
- フロントエンドでは現時点でプロジェクト一覧、作成、詳細表示を実装し、更新・削除 UI は後回しにする

### タスク

- チームメンバーは所属チームのプロジェクトにタスクを作成できる
- チームメンバーは所属チームのタスク一覧を取得できる
- チームメンバーは所属チームのタスク詳細を取得・表示できる
- チームメンバーは所属チームのタスクを編集できる
- チームメンバーは所属チームのタスクを削除できる
- タスクには担当者を設定できる
- 担当者は同じチームに所属するユーザーのみ設定できる
- タスクにはステータスを設定できる
- タスクには優先度を設定できる
- タスクには期限を設定できる
- 他チームのタスクにはアクセスできない
- フロントエンドではタスク作成、詳細表示、編集、カンバン反映を実装済みとし、削除 UI は後回しにする

### コメント

- チームメンバーは所属チームのタスクにコメントを作成できる
- チームメンバーは所属チームのタスクのコメント一覧を取得・表示できる
- コメント作成時、フロントエンドは `content` のみ送信し、`user_id` は送信しない
- 他チームのタスクコメントにはアクセスできない
- MVP ではコメントの編集と削除は対象外とする

### カンバン表示

- チームメンバーは所属チームのプロジェクト内タスクをステータス別に取得できる
- カンバン表示用 API は、ステータスごとにタスクをグルーピングして返す
- フロントエンドはプロジェクト詳細画面で `todo`、`in_progress`、`review`、`done` の 4 列カンバンを表示する
- タスクの status を編集した場合、更新後のタスクをカンバン内の正しいカラムへ反映する
- ドラッグアンドドロップは後回しにする

### 自分のタスク一覧

- ログインユーザーは自分に割り当てられたタスク一覧を取得・表示できる
- ログインユーザーは自分が作成したタスク一覧を取得・表示できる
- 取得対象は、自分が所属するチーム内のタスクのみとする
- status、priority、due_on_from、due_on_to で絞り込める
- フロントエンドの `/my/tasks` 画面は、自分が作成したタスク一覧を表示する
- フロントエンドはタスク名、status、priority、due_on、プロジェクト名、チーム名、作成者、担当者を表示する
- Task カードをクリックすると対象プロジェクト詳細へ遷移する

## 権限

MVP では TeamMember に以下の role を持たせます。

| role | 目的 |
| --- | --- |
| owner | チーム作成者。チーム管理の最上位権限 |
| admin | メンバー管理ができる管理者 |
| member | 通常メンバー |

### 権限ルール

- owner はチーム削除ができる
- owner はメンバー追加・削除ができる
- owner は admin / member 間の role 変更ができる
- owner は自分自身の role 変更と、他メンバーへの owner 付与はできない
- owner はプロジェクト、タスク、コメントを管理できる
- admin はメンバー追加ができる
- admin は role 変更ができない
- admin はプロジェクト、タスク、コメントを管理できる
- admin は owner の削除、owner の降格、owner への変更はできない
- member はチーム、プロジェクト、タスクを閲覧できる
- member はタスク作成・編集ができる
- member はコメント作成ができる
- member はメンバー管理ができない
- 他チームのデータ操作は role に関係なく禁止する

## HTTP ステータス方針

- 未ログインの場合は 401 Unauthorized を返す
- 存在しないリソース、またはログインユーザーが所属していない他チームのリソースは 404 Not Found を返す
- 所属チーム内のリソースだが、role 権限が足りない場合は 403 Forbidden を返す
- バリデーションエラーは 422 Unprocessable Entity を返す

## セキュリティ要件

- パスワードは平文保存しない
- 認証が必要な API では必ず Authorization ヘッダーの JWT を検証する
- current_user を基準にデータ取得と認可を行う
- チーム配下のリソースは必ずログインユーザーの所属チームにスコープする
- URL の ID だけでレコードを取得しない
- Strong Parameters で許可した属性のみ更新する
- before_action で認証、親リソース取得、認可を共通化する
- 権限チェックは private メソッドまたはポリシー層に整理する
- `user_id`、`team_id`、`created_by_id` などをフロントエンドから信用しない
- Task の担当者は対象チーム内ユーザーだけを許可する
- Brakeman で脆弱性チェックを行う

## 品質要件

- モデルに適切なバリデーションを定義する
- DB に NOT NULL、default、外部キー、一意制約を適切に定義する
- role、task status、task priority などは Rails 側で enum または validation を使用する
- リクエストスペックで認証、認可、正常系、異常系を検証する
- モデルスペックでバリデーションと関連を検証する
- RuboCop でコードスタイルを統一する
- 重要な処理は private メソッドで整理し、コントローラを肥大化させない
- 他チームのデータへアクセスできないことをテストで保証する
- フロントエンドは TypeScript 型を API レスポンスに合わせ、build と lint を通す
- ローディング、エラー、空状態、送信中状態を画面に用意する

## MVP 対象外・後回し

- チーム更新、削除 UI
- チームメンバー権限変更、削除 UI
- プロジェクト更新、削除 UI
- タスク削除 UI
- カンバンのドラッグアンドドロップ
- AI 機能
- 通知機能
- 招待メール送信
- ファイル添付
- タスク検索の高度な全文検索
- コメント編集・削除
- 課金
- 外部サービス連携

## 受け入れ基準

- ユーザー登録、ログイン、ログアウト、ログインユーザー取得ができる
- devise-jwt による Bearer Token 認証ができる
- ログインユーザーは自分の所属チームのみ取得できる
- 他チームのプロジェクト、タスク、コメントにアクセスできない
- チーム内でプロジェクトとタスクを作成・閲覧できる
- タスクの担当者、ステータス、優先度、期限を管理できる
- タスクにコメントを作成し、一覧取得できる
- カンバン表示用にタスクをステータス別で取得できる
- 自分の担当タスク一覧を取得できる
- 自分が作成したタスク一覧を取得できる
- フロントエンドから認証、Team、Project、Task、Comment、作成したタスクの主要画面を操作できる
- RSpec が主要な認証・認可・バリデーションを検証している
- RuboCop と Brakeman を実行できる
- Frontend build と lint を実行できる
